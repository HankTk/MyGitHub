Add Alert Conditions

An alert is generated when a condition set by you is met by the sensor or an asset.  

Example: You may want to generate an alert when the value from the sensor 'BOOSTER_PUMP_DISCHARGE_PRESSURE_HIGH' is greater than 100.

You can also combine multiple conditions and values using various operators and numerical values.

To set the conditions:

  1. Once the alert details are completed from the left pane, hover over to the right pane of the alert create window.

  1. Build your own custom conditions using the following:

  • Search - Allows you to search and select the various sensors to add to the alert condition. Once a sensor is selected, it appears in the grid below the operators. See the following image for reference.

To add this to the condition, click or drag-drop the sensor to add to the conditions grid below.

Similar to above, you can add multiple sensors to the conditions grid as shown in the image below:

  • Arithmetic and conditional operators grid - This grid allows you to add various arithmetic operators and conditional operators to the condition. Click or drag-drop to add them to the conditions grid.

For detailed information on how to build a condition using these operators, refer to

Operator Description Examples

+

Addition operator

(ST_LO_TANK_LEVEL_HIGH + ST_FWD_SEAL_TANK_LEVEL_LOW)

-

Subtraction operator

(ST_LO_TANK_LEVEL_HIGH + ST_FWD_SEAL_TANK_LEVEL_LOW) - 46

*

Multiplication operator

(ST_LO_TANK_LEVEL_HIGH *ST_FWD_SEAL_TANK_LEVEL_LOW)

/

Division operator

(ST_LO_TANK_LEVEL_HIGH + ST_FWD_SEAL_TANK_LEVEL_LOW)/2

%

Percentage operator gives the reminder in the calculation

(ST_LO_TANK_LEVEL_HIGH %10) + 45

>

Greater-than operator

(ALARM_ACK >LCARM_ALARM_FOR_TM)

<

Less-than operator

(ALARM_ACK <LCARM_ALARM_FOR_TM)

(

Open parentheses for sub condition

 

)

Closed parentheses for sub condition

 

=

Equal to operator to check if two tags are equal. Use == in the syntax to compare

(ST_LO_TANK_LEVEL_HIGH *ST_FWD_SEAL_TANK_LEVEL_LOW)==110

!=

Not equal to operator to check if two tags are not equal

(ST_LO_TANK_LEVEL_HIGH *ST_FWD_SEAL_TANK_LEVEL_LOW)!=110

,

Comma separator for multiple tags

(ALARM_ACK ,LCARM_ALARM_FOR_TM),SECA_SYSTEM_ALARM_RESET ..)

*|

OR condition for tags

((ST_LO_TANK_LEVEL_HIGH *2) | (ST_FWD_SEAL_TANK_LEVEL_LOW*4) ==220

*&

AND condition for tags

(ST_LO_TANK_LEVEL_HIGH *2) & (ST_FWD_SEAL_TANK_LEVEL_LOW*4) ==220

average

average of tags

  • average(T_LO_TANK_LEVEL_HIGH,ST_FWD_SEAL_TANK_LEVEL_LOW)

  • average( T_LO_TANK_LEVEL_HIGH,245)

sum

Sum of tags

  • sum(T_LO_TANK_LEVEL_HIGH,ST_FWD_SEAL_TANK_LEVEL_LOW)

  • sum(T_LO_TANK_LEVEL_HIGH, 245)

previous

Previous value of the selected tag

previous(T_LO_TANK_LEVEL_HIGH) +245. Here, the previous value to the current value of the sensor is added with 245.

* | and & operators should be performed between two Boolean conditions like (a>0 | b >10)

* When performing |operation, make sure that the syntax is in (a>b) | (b>d) format and not in ((a>b) | (b>d))

  • Numbers - The numbers grid allows you to add numerical values to your condition. You can click, drag-drop or input using your keyboard to add them to the conditions grid.

Click Clear All, if you want to clear the elements from the conditions grid

A condition built using the above method is given below:

Average (ST_AFT _BEARING_TEMP, ST_FWD_BEARING_TEMP) > 0.1905*ME RPM+ 46.045

Here the system calculates the average of both ST_AFT _BEARING_TEMP and ST_FWD_BEARING_TEMP sensor tags and checks if it is greater than the value of the remaining expression, that is 0.1905*ME RPM+ 46.045. If yes, an alert is generated. Else, no alert is generated.

If you wish to add an additional condition (sensor, operator or number) before any element in the condition expression, drag-drop the condition from the corresponding area on top of the expression element in the conditions grid.
